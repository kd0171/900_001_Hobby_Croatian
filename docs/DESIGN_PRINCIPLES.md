# 設計思想：クロアチア語旅行アプリ

## 1. 目的

このアプリは、日本語話者が夏のクロアチア旅行で使う初級表現を、シーン別に学ぶためのPlotly Dashアプリです。

一般的な文法学習ではなく、以下のような旅行行動に直結した表現を中心にしています。

- 海に行く
- 船に乗る
- スーパーで買い物する
- 魚市場で量り売りの魚を買う
- 魚を捌いてもらう
- 生食可否を確認する
- 観光地で道や時間を聞く

## 2. 基本方針

### 2.1 JSON教材中心

SQLiteなどのDBは使わず、教材はすべてJSONで管理します。

理由は次の通りです。

- 語彙・例文・対話を手作業またはプロンプトで追加しやすい
- アプリ本体を変更せずに教材を増やせる
- GitHub等で差分管理しやすい
- 小規模な語学教材ではRDBより構造が見えやすい

### 2.2 ファイル追加型アーキテクチャ

以下のフォルダにJSONを追加すると、自動的に読み込まれます。

```text
data/vocabulary/*.json
data/sentences/*.json
data/dialogues/*.json
data/references/*.json
```

既存の単体ファイルも読み込みます。

```text
data/vocabulary.json
data/sentences.json
data/dialogues.json
```

## 3. UI方針

### 3.1 スマホ優先

旅行中にスマホで使う可能性が高いため、以下を重視します。

- タイトルは短くする
- 長い説明は折り畳む
- 設定項目はAccordionに入れる
- 表は横スクロール可能にする
- Dropdownが親カードで隠れないようにする

### 3.2 学習履歴は未実装

現段階ではUIと教材拡張を優先し、学習履歴は保存しません。

将来的に必要になった場合は、以下の順で検討します。

1. ブラウザ内の `dcc.Store`
2. localStorage
3. ユーザー別JSON
4. SQLiteやPostgreSQL

## 4. 教材設計

### 4.1 全教材にsceneを持たせる

各単語・例文・対話には `scene` を入れます。

例：

```json
{
  "scene": "fish_market",
  "level": 2,
  "tags": ["seafood", "cutting"]
}
```

これにより、チェックボックスでシーンごとに絞り込めます。

### 4.2 カタカナ発音

各単語・文には `pronunciation_ja` を持たせます。

カタカナは正確な発音表記ではなく、日本語話者向けの補助です。

### 4.3 魚介類の日本語説明

brancin や orada のような魚名は、日本の魚名と完全には対応しません。
そのため、単純なカタカナ表記ではなく、以下のように説明します。

- brancin / lubin = ヨーロピアンシーバス（スズキに似た白身魚）
- orada = ヨーロッパヘダイ（タイに近い白身魚）
- arbun = パンドラ／タイ科の小型白身魚

## 5. 安全・文化メモ

魚市場や海産物の表現では、次の注意を入れる方針です。

- 生食できるかは必ず店員に確認する
- 魚市場の魚が常に刺身向けとは限らない
- ウニや貝類の採集は場所・規則・保護区域に注意する
- 日本語の「刺身」とクロアチア語の `sirovo` は完全に同じ概念ではない
